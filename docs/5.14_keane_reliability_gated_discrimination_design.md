# 5.14 Keane Reliability-Gated Discrimination Design

## 목적
- Keane 코호트(ds003404+ds005073)에서 BS-NET의 역할을 **질환 분류기**가 아닌 **reliability scorer / QC gate**로 검증한다.
- 질문: `rho_hat_T` 기반 사전 선별(또는 가중치)이 BP vs SZ discrimination 성능을 유의하게 개선하는가?

## 핵심 개념
- BS-NET 출력 `rho_hat_T`는 측정 신뢰도 추정치이다.
- 분류 모델은 별도(예: logistic_l2, linear_svm)로 유지한다.
- 따라서 파이프라인은 `Data -> reliability gate -> classifier` 구조로 정의한다.

## 사전 정의 가설
- Primary (confirmatory):
  - 설정: `feature=fc_bsnet_pred`, `model=logistic_l2`, `gate_mode=hard`, `rho_quantile=0.4` (상위 60%)
  - endpoint: BalAcc
  - 검정: permutation p-value
- Exploratory:
  - baseline(`gate_mode=none`)
  - hard-gating grid (`rho_quantiles={0.3,0.4,0.5}`)
  - soft-weight grid (`rho_gammas={0.5,1.0,2.0}`)
  - 필요 시 feature/model 조합 확장

## 데이터 누수 방지 (필수)
- Hard gate threshold는 **각 CV fold의 train set에서만** 계산한다.
- 동일 threshold를 해당 fold의 test set에 적용한다.
- 전 데이터에서 threshold를 미리 계산해 CV에 적용하는 방식은 금지한다.

## Split validity 규칙
- gate 적용 후 train/test 모두에서 양 클래스가 유지되어야 한다.
- 기본 하한: fold 내 클래스별 최소 표본 `n >= 10`.
- 미충족 split은 skip하고, `n_splits_used`, `coverage`를 함께 보고한다.

## 통계 프레임
- Confirmatory family:
  - primary 1개 검정 (필요 시 Holm 보정 가능)
- Exploratory family:
  - 다중 설정의 permutation p-value를 BH-FDR로 보정
- 보고:
  - BalAcc/AUC/AUPRC mean±std
  - coverage, n_eval_subjects, n_splits_used
  - raw p, Holm/FDR adjusted p(or q)

## 해석 규칙
- 성능 개선 + 통계 유의:
  - `rho_hat_T`가 분류 정보가 있는 샘플을 선별하는 데 기여 가능
- 성능 비개선/비유의:
  - 신뢰도 향상이 질환 분리 신호 향상으로 자동 전이되지 않음을 의미
  - BS-NET의 임상적 위치는 discrimination model이 아니라 reliability estimator/QC tool로 유지
