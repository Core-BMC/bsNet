## 0. 코딩 컨벤션 (PEP 8 기반)

본 리팩토링의 모든 코드는 아래 규칙을 준수한다.

**네이밍**
- 모듈/패키지: `snake_case` (예: `graph_metrics.py`)
- 함수/변수: `snake_case` (예: `get_fc_matrix`, `n_rois`)
- 클래스: `PascalCase` (예: `BSNetConfig`)
- 상수: `UPPER_SNAKE_CASE` (예: `DEFAULT_N_BOOTSTRAPS`, `PALETTE`)
- Private: `_leading_underscore` (내부 헬퍼)

**포매팅**
- 들여쓰기: 4 spaces (탭 금지)
- 최대 줄 길이: 88자 (Black 기본값, PEP 8 79자에서 완화)
- import 순서: stdlib → third-party → local (`isort` 적용)
- 빈 줄: 최상위 함수/클래스 사이 2줄, 메서드 사이 1줄
- 후행 공백/쉼표 제거

**Type Hints (Python 3.9+)**
```python
def get_fc_matrix(
    time_series: np.ndarray,
    vectorized: bool = True,
    use_shrinkage: bool = False,
) -> np.ndarray:
```

**Docstring (Google Style)**
```python
def spearman_brown(r_t: float, k: float) -> float:
    """Apply the Spearman-Brown prophecy formula.

    Extrapolates split-half reliability to full-length test reliability.

    Args:
        r_t: Observed split-half correlation coefficient.
        k: Ratio of target length to observed length (T/t).

    Returns:
        Predicted full-length reliability coefficient, clipped to [0, 1].

    Raises:
        ValueError: If r_t is not in (0, 1) range.
    """
```

**Linter/Formatter 도구** (`pyproject.toml`에 설정 추가)
```toml
[tool.black]
line-length = 88
target-version = ["py39"]

[tool.isort]
profile = "black"

[tool.ruff]
line-length = 88
select = ["E", "F", "W", "I", "N", "UP", "B", "SIM"]
target-version = "py39"
```

**기타 PEP 8 준수 사항**
- `==` 비교 시 `None`은 `is None`/`is not None` 사용
- Bare `except:` 금지 → 구체 예외 타입 명시 (`except ValueError:`)
- Mutable default argument 금지 (`def f(x=[])` → `def f(x=None)`)
- f-string 우선 사용 (`format()`, `%` 지양)
- Wildcard import 금지 (`from module import *`)

## 3. 리팩토링 계획

### Phase A: 공용 유틸리티 추출 (우선순위 1)

**목표**: 중복 코드 제거, 단일 진실 원천(Single Source of Truth) 확립

1. **`src/core/config.py` 신설** — 전역 상수/하이퍼파라미터 중앙 관리
   ```python
   @dataclass(frozen=True)
   class BSNetConfig:
       # Simulation
       n_rois: int = 400
       tr: float = 1.0
       short_duration_sec: int = 120
       target_duration_min: int = 15
       noise_level: float = 0.25
       ar1: float = 0.6
       # Bootstrap
       n_bootstraps: int = 100
       reliability_coeff: float = 0.98
       empirical_prior: tuple[float, float] = (0.25, 0.05)
       # Graph
       fc_density: float = 0.15
       # Paths
       artifacts_dir: str = "artifacts/reports"
       figure_dir: str = "docs/figure"
   ```

2. **`src/core/pipeline.py` 신설** — bootstrap loop를 단일 함수로 추출
   ```python
   def run_bootstrap_prediction(
       short_obs: np.ndarray,
       fc_reference: np.ndarray,
       config: BSNetConfig
   ) -> BootstrapResult:
       """scripts 4개 + visualization 2개에서 반복되던 bootstrap loop 통합."""
   ```

3. **`src/visualization/style.py` 신설** — 통합 스타일 정의
   ```python
   PALETTE = {"bsnet": "#4A90E2", "raw": "#E27396", "true": "#2c7bb6", ...}
   def apply_bsnet_theme(ax): ...
   def save_figure(fig, name, config): ...  # artifacts + docs 이중저장 통합
   ```

4. **`src/core/graph_metrics.py` 신설** — 그래프 분석 함수 통합
   - `threshold_matrix()`, degree variance, small-worldness, community detection
   - figure3, figure4에서 중복 정의된 함수 통합

### Phase B: 스크립트 정리 (우선순위 2)

1. **MoBSE 의존성 분리**: `sys.path.append` 제거 → `pyproject.toml`에 optional dependency로 선언하거나 환경변수(`MOBSE_PATH`)로 전환
2. **scripts/ → CLI entry_points**: `pyproject.toml`에 `[project.scripts]` 섹션 추가
   ```toml
   [project.scripts]
   bsnet-simulate = "scripts.run_synthetic_baseline:run_simulation"
   bsnet-sweep = "scripts.sweep_simulation:run_duration_sweep"
   bsnet-validate = "scripts.run_real_data_scale:run_scale_up_pipeline"
   ```
3. **logging 전환**: `print()` → `logging.getLogger(__name__)` 적용
4. **seed 관리 체계화**: `BSNetConfig.seed` 필드 추가, 각 스크립트에서 config 기반 초기화

### Phase C: 시각화 통합 (우선순위 3)

1. **figure4 계열 통합**: `plot_figure4_subnetworks.py`를 메인으로 두고, broken_axis와 yeo100_formatter를 내부 옵션(파라미터)으로 흡수
2. **개인 경로 제거**: plot_figure1의 `/Users/hwon/.gemini/...` 삭제
3. **입력 검증 추가**: CSV 로드 시 필수 컬럼 확인, 누락 시 명시적 에러
4. **컬러 팔레트 통일**: `style.py`의 중앙 팔레트를 모든 figure에서 import

### Phase D: 프로젝트 정리 (우선순위 4)

1. **`build/lib/` 처리 (setuptools 빌드 캐시)**
   - `build/lib/`은 `pip install -e .` 시 생성된 src/ 스냅샷임
   - diff 결과: 10개 중 8개 동일, 2개(`plot_figure3`, `plot_figure4_subnetworks`)는 src/가 최신
   - build/lib 구버전은 `data_loader.get_fc_matrix` import를 사용하는 깔끔한 구조였으나, src/에서 Schaefer 400 스케일업 시 로컬 함수로 재정의됨 (Phase A에서 원복 예정)
   - **조치**: `build/` 전체를 `.gitignore`에 추가 후 삭제. 재빌드 시 자동 생성됨
2. `archieve/` → 삭제 (빈 디렉토리, 오타)
3. `backup/` → 필요시 git history로 대체, 삭제 검토
4. Type hints 추가 (Python 3.9+ 문법)
5. Google Style docstring 보강
6. `tests/` 디렉토리 신설 — core 모듈 단위 테스트
