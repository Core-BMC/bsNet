### Design Decisions

1. **BS-NET utility reframing**
   - Edge-level FC correction (bootstrap mean, edge-wise attenuation) → FC 개선 실패
   - ρ̂T는 per-subject reliability estimate → downstream quality predictor
   - Proof: tertile stratification에서 monotonic dose-response 확인
   - 논문 프레이밍: "ρ̂T enables principled quality control of short-scan FC data"

2. **Reference 길이 5min 채택**
   - 4min vs 5min: 동일 N=399 (KKI max=220s ↔ Peking min=344s 사이 gap)
   - Noble et al. 2019 및 ABIDE 기준과 일치

3. **ADHD-200 S3 다운로드 실패 → NITRC tar.gz**
   - ADHD-200 PCP는 ABIDE PCP와 S3 구조가 다름 (Outputs/ 패턴 없음)
   - NITRC에서 ADHD200_CC200_TCs_filtfix.tar.gz (275MB) 수동 다운로드

### New Idea: τ_min (Minimum Common Bootstrap Duration)

**개념**: "2분"은 arbitrary가 아니라 hemodynamic low-freq signal의 최소 공집합 cycle에 근거.
- rsfMRI 주요 대역: 0.01–0.1 Hz → 가장 느린 성분 1 cycle = 100s
- Block bootstrap 최소 조건: ~2 cycles ≥ 2/f_low → ~200s (보수적) 또는 ~120s (실용적)
- 수식: τ_min = n_blocks × l_block / f_s ≥ 2/f_low

**BS-NET과의 연결**:
- ρ̂T(τ) 곡선에서 plateau에 도달하는 최소 τ를 τ_min으로 정의
- Duration sweep 데이터로 empirically 추정 가능
- Network-specific 분석 불필요 — aggregate reliability (ρ̂T)로 충분
- 코호트/조건별 τ_min 가변성 인정하면서 ~2min을 일반적 하한으로 제안

**논문 위치**: Discussion에서 "Why ~2 minutes?" 에 대한 principled answer 제공

### Design Decisions

1. **Duration sweep 원본 설계의 결함 발견 및 폐기**
   - 원래: τ_short 변화 → τ_ref = total − τ_short (역비례)
   - 문제: short FC 개선(↑)과 reference FC 악화(↓) 효과가 상충
   - r_FC(τ_short=300, τ_ref=600)와 ρ̂T(τ_short=120, τ_ref=780)는 서로 다른 duration의 reliability → 비교 불가

2. **올바른 수렴 실험 설계: τ_short 고정, τ_ref 변화**
   - ts_short = ts[0:short_vols] — 항상 동일한 구간
   - ts_ref = ts[short_vols : short_vols+ref_vols] — 점점 길어짐
   - 단일 변수(τ_ref)만 변화 → 깔끔한 수렴 곡선
   - r_FC(τ_ref) → ρ̂T as τ_ref → ∞ 직접 증명

3. **2-phase 분리로 계산 효율 극대화**
   - r_FC는 bootstrap 불필요 (deterministic FC correlation) → 수 분
   - BS-NET ρ̂T는 τ_short당 1회만 (최대 reference 사용) → 기존 대비 1/20
   - 총 소요: ~30min (기존 duration sweep ~450h 대비 ~900× 빠름)

4. **BLAS thread oversubscription 해결**
   - **증상**: 서버 CPU 24코어 모두 100%, `--n-jobs 8` 임에도 비정상적으로 느림
   - **원인**: NumPy/SciPy가 내부적으로 OpenBLAS/MKL multi-threading 수행 → 8 worker × N BLAS threads = 수백 개 스레드 경합
   - **해결**: BLAS threading을 1로 고정 후 Python worker 수를 늘림
     ```bash
     export OMP_NUM_THREADS=1 MKL_NUM_THREADS=1 OPENBLAS_NUM_THREADS=1 NUMEXPR_NUM_THREADS=1
     python src/scripts/run_convergence_validation.py --n-jobs 20
     ```
   - **효과**: 24코어 서버에서 총 소요 시간 ~30min → ~5min 수준으로 단축
   - **일반 원칙**: `ProcessPoolExecutor` + NumPy 조합 시 항상 BLAS threads=1 설정 필수

#### Convergence Validation — Final Results (N=49, 100bs × 10seeds)
| τ_short | ρ̂T (mean±SD) | r_FC peak | Gap (ρ̂T−r_FC) | Per-subj gap positive |
|---------|--------------|-----------|----------------|----------------------|
| 60s     | 0.756±0.032  | 0.566 (300s) | 0.190       | 49/49 (100%)         |
| 120s    | 0.771±0.032  | 0.638 (240s) | 0.134       | 25/25 (100%)         |

**핵심 발견**:
1. **r_FC < ρ̂T 항상 성립**: 모든 유효한 (τ_short, τ_ref) 조합에서 per-subject gap 100% positive
2. **Non-stationarity**: r_FC는 τ_ref≈240-300s에서 peak 후 하락 (temporal distance 효과)
3. **Gap 축소 패턴**: τ_short 증가 시 gap 감소 (60s: 0.190 → 180s: 0.112) — 더 긴 short scan은 더 reliable
4. **ρ̂T 안정성**: τ_short 60-180s에서 ρ̂T = 0.756-0.771 (range 0.015) — subject의 true reliability를 일관되게 추정

#### τ_min Empirical Estimation (18-point fine-grained)
| τ_short | ρ̂T | seed SD | % of peak |
|---------|-----|---------|-----------|
| 30s  | 0.709 | 0.011 | 91.6% |
| 40s  | 0.741 | 0.006 | 95.7% ← 95% threshold |
| 50s  | 0.750 | 0.006 | 97.0% |
| 60s  | 0.756 | 0.005 | 97.7% |
| 90s  | 0.767 | 0.005 | 99.2% ← plateau 진입 |
| 120s | 0.771 | 0.004 | 99.7% |
| 150s | **0.774** | 0.003 | **100%** ← peak |
| 180s | 0.769 | 0.003 | 99.3% |
| 240s | 0.749 | 0.002 | 96.9% ← ref artifact 시작 |
| 300s | 0.712 | 0.002 | 92.0% |
| 450s | 0.708 | 0.003 | 91.6% |

**τ_min 해석**:
- 95% of peak 기준: τ_min ≈ 40s (매우 보수적)
- Plateau 진입 기준 (Δρ̂T < 0.005/step): τ_min ≈ 90s
- 실질적 권장: **60–120s** (97-99% peak, seed SD < 0.005)
- Hemodynamic 이론: 0.01 Hz × 2 cycles = 200s → 이론적 상한과 plateau 일치

### Design Decisions

1. **τ_min 정의 기준 선택**
   - 95% of peak (τ_min=40s): too lenient, 아직 상승 구간
   - Plateau flatness (Δ < 0.005): τ_min=90s, 이론과 일치
   - 논문에서는 두 기준 모두 제시하되, 실용적 권장 60-120s 채택

2. **Non-stationarity의 의미**
   - r_FC 하락은 BS-NET의 문제가 아니라 rsfMRI의 본질적 한계
   - ρ̂T는 non-stationarity를 bypass하여 ideal reliability 추정
   - 이것이 "simple split-half보다 BS-NET이 우수한 이유" — Discussion 핵심 논점

### Fig 1A Panel A — Pipeline Schematic Illustration 지시사항

> 최종본은 사용자가 외부 도구(Illustrator/PowerPoint/Figma)로 제작하여 교체함.
> 아래는 Panel A의 디자인 사양 및 재현 지침.

**구성**: 좌→우 5단계 pipeline flow

| 단계 | 라벨 | 시각 요소 | 수식/주석 |
|------|------|-----------|-----------|
| 1. Input | Short Scan FC (2 min) | 반투명 brain connectome (noisy/sparse edges, amber 톤) | `T_short` |
| 2. Shrinkage | Ledoit-Wolf Shrinkage | FC 행렬 heatmap (blue gradient, regularized) | `Σ̂ = (1−λ)Σ̂ₛ + λF` |
| 3. Bootstrap | Block Bootstrap | 시계열 리샘플링 블록 다이어그램 (stacked layers) | `N = {S¹, ..., Sⁿ}` |
| 4. Extrapolation | Spearman-Brown | FC 행렬 스택 → 단일 행렬 수렴 | `ρ̂ = k·r / (1+(k−1)r)` |
| 5. Stabilization | Bayesian Prior | Prior + Posterior 결합 (grid/matrix overlay) | `ρ_post ∝ P(D|ρ)P(ρ)` |
| 6. Output | Reliability-Adjusted ρ̂T Matrix | 선명한 brain connectome (dense edges, green 톤) | Fisher z Attenuation: `ρ̂ = tanh(z − tanh⁻¹(ρ_boot))` |

**색상 체계**:
- Input: Amber (#fdae61) — noisy, sparse
- Processing stages: Blue gradient (#4A90E2 → #5BA3F5)
- Output: Green (#2ecc71) — clean, reliable
- 화살표: Gray (#555555), 굵기 2pt

**Typography**:
- 단계 이름: 11pt bold, white on colored background
- 수식: 9pt italic, below each stage
- 연결 화살표: 각 단계 사이 → 기호

**크기**: 가로 전폭 (figure width 16"), 세로 ~3" (전체 figure의 상단 1/5)

**Brain connectome 스타일**:
- 3D 렌더링 brain (lateral view, semi-transparent cortex)
- Input brain: sparse orange connections, low density → "noisy short scan"
- Output brain: dense green connections, high density → "reliability-adjusted"
- MNI space 기준, Schaefer 200 parcellation nodes

**추가 디자인 노트**:
- 각 단계를 rounded box 또는 card 스타일로 감싸되, brain 이미지는 box 밖으로 약간 돌출 가능
- 전체 배경: light gray (#F7F9FC) 또는 white
- B1–B3, C 패널과의 시각적 연속성 유지 — 같은 blue/amber 색상 팔레트 사용
