# 5.13 ADHD Discrimination Citation Guide (for BS-NET)

> 목적: “잘 인용되고 방어 가능한 분석법”으로 ADHD vs HC discrimination을 수행하고, 이를 BS-NET(ρ̂T) 맥락에 정합적으로 연결한다.

---

## 1) 권장 분석 프레임 (한 줄 요약)

**Tangent FC + non-sparse linear classifier (L2-Logistic 또는 Linear SVM) + site/confound 통제 + permutation test**를 기본축으로 사용한다.

이 선택의 핵심 근거:
- connectome 예측 벤치마크에서 tangent representation과 선형모델 조합이 일관적으로 우수함.
- ADHD-200 계열에서는 site/개인특성(confound) 영향이 커서 이를 명시적으로 통제해야 과대추정을 피할 수 있음.
- 다기관(multi-site) FC 분석에서 harmonization(ComBat) 적용이 site effect 완화에 유효함.

---

## 2) 본문에 바로 넣는 Methods 문장 템플릿

### 2.1 Core pipeline (Methods)

본 연구의 ADHD vs HC 분류는 resting-state functional connectome 기반의 표준 예측 파이프라인(ROI 정의 → connectome 추정 → 선형 분류기)을 따랐다. Connectome 표현은 correlation/partial correlation/tangent를 비교하되, 1차 분석(primary analysis)은 다중 코호트 벤치마크에서 재현성 있게 권장된 tangent-space embedding을 사용하였다. 분류기는 비희소 선형모델(L2-regularized logistic regression 또는 linear SVM)을 채택하였다.

### 2.2 Multi-site control (Methods)

ADHD-200 데이터의 다기관 이질성(site, demographics, acquisition variability)을 고려하여, (i) 연령/성별/site를 공변량으로 포함하거나, (ii) fold 내부에서 site harmonization(ComBat)을 적용하였다. 모든 전처리/하모나이제이션/모델 선택 절차는 데이터 누수를 방지하기 위해 학습 폴드에서만 적합하고 검증 폴드에 변환을 적용하였다.

### 2.3 BS-NET integration (Methods)

BS-NET 적용 실험에서는 raw FC 대신 BS-NET corrected FC를 동일한 분류 파이프라인에 투입하였다. 추가적으로 subject-level reliability(ρ̂T)를 표본 가중치로 사용한 가중 분류(예: \(w_i=\rhôT_i^\gamma\), \(\gamma \in [1,2]\))를 보조 분석으로 수행하여 reliability-aware 학습의 효과를 평가하였다.

### 2.4 Statistical testing (Methods)

성능 평가는 balanced accuracy, AUROC, AUPRC를 보고하였고, 클래스 불균형 및 임의 정렬 효과를 통제하기 위해 permutation test를 병행하였다. 교차검증 기반 성능의 과대추정 위험을 줄이기 위해 site-generalization 평가(예: leave-one-site-out)를 우선 보고하였다.

---

## 3) Results/Discussion 문장 템플릿

### 3.1 보수적 결과 기술

“BS-NET은 ADHD vs HC 분류에서 대규모 정확도 향상(clinical-grade gain)을 주장하기보다, 짧은 스캔에서 얻은 FC의 신뢰도 정보를 정량화하고(reliability indexing), site/confound를 통제한 조건에서 분류 성능의 안정성을 유지하는 역할을 보였다.”

### 3.2 유의성 미확보 시 문구

“Permutation 기반 검정에서 일관된 유의수준에 도달하지 못했으므로, 본 결과는 exploratory evidence로 해석해야 하며 임상 진단 바이오마커의 확증적 근거로 해석해서는 안 된다.”

### 3.3 왜 이 설계가 방어 가능한가

“본 분석은 tangent-linear 표준 조합, 다기관 site effect 보정, 누수 방지형 cross-validation, permutation 유의성 검정을 결합하여 ADHD MRI 분류 연구에서 반복적으로 지적된 과대추정 위험을 최소화하도록 설계되었다.”

## 6) 실행 스크립트 연결 (현재 bsNet 코드베이스 기준)

- 분석: `src/scripts/run_reliability_aware_classification.py`
- 기존 탐색 분석: `src/scripts/run_reliability_aware_clustering.py`
- 시각화: `src/visualization/plot_patient_utility_clustering.py`

권장 CLI 예시(사용자 실행):
```bash
source .venv/bin/activate
export OMP_NUM_THREADS=1 MKL_NUM_THREADS=1 OPENBLAS_NUM_THREADS=1 NUMEXPR_NUM_THREADS=1

# (예정) supervised discrimination
python3 -m src.scripts.run_reliability_aware_classification \
  --balance-classes \
  --n-repeats 20 \
  --n-permutations 1000 \
  --n-jobs 8 \
  --primary-fc tangent \
  --model logistic_l2
```

> 참고: `run_reliability_aware_classification.py`는 LOSO/Stratified 평가, class-balance 보정, rho 가중치, permutation 옵션을 지원한다.
