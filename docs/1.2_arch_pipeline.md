    **1. 논문의 핵심 내용 요약**

    **1.1 부트스트랩을 이용한 시뮬레이션 데이터 생성**
    **데이터 생성 과정(DGP) 정의** 
    논문은 fMRI 데이터의 생성 과정을 **data‐generating process (DGP)** 로 정의하며, 이는 임의 변수 Y로부터 샘플 y를 생성하는 메커니즘이다. 
    실제 뇌영상의 DGP는 미지이기 때문에 연구자는 이를 근사화한 모델 \hat{f}를 사용한다. 
    부트스트랩은 실제 데이터 샘플 y를 사용하여 \hat{f}를 추정하는 일반적인 통계기법으로, 
    이는 시뮬레이션 모델의 파라미터를 실제 데이터에서 추정하여 새로운 데이터를 재생산하는 방식이다.
    **파라메트릭 부트스트랩(PBS)** 
    파라메트릭 DGP f_\theta 를 가정할 경우, 
    실제 데이터에서 추정한 파라미터 \hat{\theta}를 사용하여 시뮬레이션을 생성한다. 
    예를 들어 실험 조건(기저선, 과제반응, 생리적 노이즈, 움직임 등)에 대응하는 파라미터를 추정한 뒤, 
    해당 파라미터에 기반해 여러 샘플을 모사함으로써 **실제 데이터베이스의 통계적 다양성을 재현**한다.
    **비파라메트릭 부트스트랩 평가** 
    파라메트릭 모델의 타당성을 검증하기 위해, 
    논문은 **비파라메트릭 부트스트랩**을 사용하여 실제 데이터의 통계량 분포와 
    시뮬레이션에서 얻은 분포를 비교한다. 
    요약 통계량 s (예: 평균, 중앙값, 분위수)을 부트스트랩 반복을 통해 얻은 분포와 비교하고, 
    실제 통계량이 신뢰구간을 벗어나면 모델이 데이터 특성을 제대로 재현하지 못한다고 판단한다.

    **1.2 시뮬레이션 모델**
    **ADD(선형 혼합 모델)** 
    기저선(B), 느린 드리프트(D), 생리적 노이즈(P), 과제 관련 활성(A), 랜덤 노이즈(E) 등 선형적으로 결합된 모형이다. 
    각 구성요소의 공간·시간 분포는 실제 데이터에서 추정하며, 
    생리적 노이즈는 ICA와 CORSICA를 사용해 분리한다. 
    노이즈 분산 추정 오차를 줄이기 위해 **수축(shrinkage) 보정**을 적용한다.
    **SM(비선형 움직임 모델)** 
    ADD와 동일한 구성요소를 갖지만, **고해상도(1×1×1 mm) 볼륨**에서 기저선·생리적 노이즈·활성 신호를 합성한 뒤, 
    **강체(rigid‑body) 움직임 과정**을 적용하고 저해상도로 다시 샘플링한다. 
    이 과정은 움직임에 의해 발생하는 공간적 상관과 느린 드리프트를 시뮬레이션한다.

    **1.3 시뮬레이션 평가 결과**
    **공간 및 시간 상관 구조** 
    실제 데이터는 움직임 보정 후에도 **음의 시간적 자기상관**과 **양의 공간적 자기상관**을 갖는다. 
    ADD 모델은 시간적 자기상관을 질적으로는 재현했지만, 
    **공간적 자기상관이 매우 낮아 현실적이지 않았다**는 점이 지적되었다. 
    SM 모델은 움직임을 모사함으로써 공간적 상관을 높여 실제 데이터와 더 유사한 분포를 보였지만, 
    여전히 완전한 일치는 아니었다.
    **PCA 분산** 
    PCA 분석에서 ADD 시뮬레이션의 잔차는 거의 균일한 분산을 보여 상관 구조가 없음을 나타냈지만, 
    SM 시뮬레이션의 잔차는 실제 데이터에 가까운 분산 곡선을 보여 잔여 상관을 더 잘 모사했다.
    **논문의 결론** 
    연구는 파라메트릭/비파라메트릭 부트스트랩을 결합한 시뮬레이션 프레임워크의 가능성을 보여주지만, 
    모델 파라미터 추정과 모델 구조의 불완전성이 결과에 영향을 미친다는 점을 강조한다. 
    또한, 가설 검정에서는 **비파라메트릭 부트스트랩**을 선호하고, 
    파라메트릭 시뮬레이션은 모델 파라미터를 통제할 수 있다는 장점이 있음을 제안한다.
            
    **2. 내 계획과 비교 시 중요한 시사점**
    **현실적 데이터 구조 모사** 
    논문은 실제 fMRI 데이터의 **공간·시간 상관 및 잡음 구조**를 시뮬레이션 모델에 반영해야 함을 강조한다. 
    내 계획에서 블록 부트스트랩으로 시계열 세그먼트를 재표본하는 것은 자기상관을 고려하지만,
    **생리적 노이즈, 느린 드리프트, 움직임** 등 여러 잡음원 분해가 필요하며, 
    **이들의 추정이 정확하지 않으면 결과가 왜곡될 수 있다.**
    **모델 파라미터 추정의 불확실성** 
    파라메트릭 부트스트랩의 성능은 파라미터 추정에 크게 의존하며, 
    추정치의 신뢰성을 평가해야 한다. 
    논문은 수축 보정(shrinkage)이 분산 추정의 안정성에 도움을 준다고 보고한다. 
    내 계획에서 신뢰도 추정치가 낮을 경우 **보정 공식이 발산**하는 문제를 언급하였는데, 
    **shrinkage 또는 베이지안 정규화를 적용해 R값을 안정화하는 것이 타당하다.**
    **비파라메트릭 부트스트랩 평가** 
    논문은 시뮬레이션 결과를 실제 데이터의 통계량 분포와 비교하는 데 비파라메트릭 부트스트랩을 사용한다. 
    내 계획도 부트스트랩 반복으로 \rho^*(T) 추정치를 얻은 뒤 
    **percentile 신뢰구간** (2.5th–97.5th)을 계산하지만, **실제 장시간 데이터와 비교 검증**하는 단계가 없다.
    **시간 비정상성(Stationarity) 검토** 
    논문에서 ADD와 SM 모델의 주요 한계는 비정상성 및 움직임으로 인한 시간적 변화였다. 
    내 계획에서도 Spearman–Brown 공식의 정상성 가정을 위배할 위험이 지적되었으므로, 
    **분석 전 시계열의 분포 안정성을 확인**하는 절차가 필요하다.
    **움직임 모사 및 공간적 상관** 
    SM 모델은 고해상도 볼륨에 움직임을 적용해 **공간적 상관과 느린 드리프트를 재현**한다. 
    내 계획에서는 네트워크를 벡터화해 상관을 계산하는데, 
    움직임에 의해 변화하는 공간적 패턴이 신뢰도 추정에 영향을 미칠 수 있다. 
    **시뮬레이션에서 움직임과 드리프트를 포함하거나,** 
    **실제 데이터 전처리에서 잔여 움직임 신호를 엄격히 제거**해야 한다.
    **양쪽 부트스트랩 결합** 
    논문은 파라메트릭 시뮬레이션(ADD, SM)과 비파라메트릭 부트스트랩 평가를 결합한다. 
    내 계획도 **블록 부트스트랩**과 **신뢰도 추정**을 결합하지만, 
    **데이터베이스 차원**의 비파라메트릭 평가를 추가하면 모델링 오류를 감지하고 보정할 수 있다.

    **3. 계획에 추가할 방법론적·이론적 대책**
    아래는 논문에서 도출한 시사점을 기반으로 내 계획을 강화하기 위한 구체적인 제안이다.
    **3.1 신뢰도 추정의 안정화**
    **shrinkage 또는 베이지안 정규화** 
    감쇠 보정에서 R_{\widehat{N}}(t)와 R_N(t)가 낮으면 보정값이 불안정해질 수 있다. 
    Bellec et al.는 모델 파라미터 추정 시 분산 추정치를 축소(shrinkage)해 안정성을 높였다. 
    유사하게, 신뢰도 추정치에 최소값(예: 0.1) 또는 베이지안 사전분포를 적용하여 **과도한 발산을 방지**할 수 있다.
    **모델 불확실성 고려** 
    예측 네트워크가 결정론적일 경우 R_{\widehat{N}}(t)=1로 가정하는 대신, 
    모델 출력의 불확실성을 추정하여 **예측 신뢰도**를 계산하는 것이 바람직하다. 
    드롭아웃이나 앙상블 기법을 적용해 입력이 동일할 때 모델 예측의 변동을 측정하고 
    이를 R_{\widehat{N}}(t)에 반영할 수 있다.

    **3.2 시간적 비정상성 및 자기상관 처리**
    **사전 안정성 검사** 
    Spearman–Brown 공식은 시계열의 통계적 특성이 일정하다는 가정에 기반한다. 
    부트스트랩을 적용하기 전, **시계열을 여러 구간으로 나누어 평균과 분산, 파워스펙트럼이 안정한지 검사**하고, 
    크게 변하는 구간은 제거하거나 별도 모델링한다.
    **블록 길이 최적화** 
    논문에서는 모델 ADD/SM이 자기상관을 제대로 모사하지 못하는 문제가 있었다. 
    내 계획의 부트스트랩은 블록 길이를 임의로 10–20 TR로 설정하지만, 
    **자기상관 함수(ACF)를 기반으로 최적 블록 길이를 선택**하거나 **circular block bootstrap**을 사용하면 
    더 안정적인 표본 분포를 얻을 수 있다.
    **split‑half 편향 교정** 
    짝/홀 TR 분할은 시간적 자기상관으로 인해 신뢰도를 과대평가한다. 
    일정 간격(lag)을 두고 샘플을 추출하거나, **prewhitening**을 통해 자기상관을 감소시킨 뒤 split‑half를 수행할 수 있다.

    **3.3 시뮬레이션 및 검증 단계 추가**
    **실제 장시간 데이터와의 비교** 
    계획의 목표는 짧은 데이터로 장시간 네트워크 상관을 예측하는 것이다. 
    Bellec et al.는 실제 데이터베이스와 시뮬레이션 결과를 비교하여 모델의 적합성을 평가한다. 
    유사하게, **일부 피험자에 대해 장시간(예: 10–20 분) rsfMRI 데이터를 확보**해 두 분류(2분과 전체 길이)로 나눈 후, 
    제안한 부트스트랩 추정치가 실제 장시간 상관과 얼마나 일치하는지 검증해야 한다.
    **시뮬레이션 기반 검증** 
    Bellec et al.가 제안한 ADD/SM 모델과 같이, 
    **정해진 reference 네트워크 구조를 갖는 시뮬레이션 데이터를 생성**하고 
    제안한 부트스트랩·감쇠 보정 방법을 적용하여 
    \widehat{\rho}^(T)*가 실제* \rho^(T)를 얼마나 잘 복원하는지 평가한다. 
    이를 통해 방법론의 바이어스와 분산을 사전에 파악할 수 있다.
    **비파라메트릭 부트스트랩 평가** 
    짧은 길이 데이터에서 추정한 
    \widehat{\rho}^{*(b)}(T)들을 이용하여 **분포와 요약 통계량**을 계산하고, 
    이를 실제 장시간 상관의 분포와 비교한다. 
    부트스트랩 신뢰구간에서 실제 값이 벗어나면 모델링이나 파라미터 추정에 문제가 있는 것이다.

    **3.4 잡음·모션 모델링 강화**            
    **생리적 노이즈와 느린 드리프트 분해** 
    Bellec et al.는 ICA(CORSICA)와 선형 회귀를 활용하여 
    느린 드리프트와 생리적 노이즈를 분리한다. 
    내 계획에서도 **fMRIPrep + XCP‑D** 파이프라인을 사용해 이러한 잡음을 최대한 제거한 뒤 네트워크를 추정해야 신뢰도 추정이 정확해진다.
    **모션 잔차 모니터링** 
    SM 모델이 보여주듯, 움직임은 공간적 상관과 느린 드리프트를 증가시킨다. 
    rsfMRI에서 움직임 파라미터와 스크러빙을 사용하여 움직임으로 인한 변동을 제거하고, 
    잔차 움직임의 분포를 점검해야 한다.

    **3.5 데이터 및 도구 활용**
    **전처리/아틀라스** 
    사용 문서에 나온 fMRIPrep, XCP‑D, Tedana, Schaefer/CAB‑NP 아틀라스 등을 활용해 표준화된 네트워크 추정 파이프라인을 구성한다. 
    이는 네트워크 벡터화 후 평균=0, 분산=1로 정규화하는 과정과 일관된다.
