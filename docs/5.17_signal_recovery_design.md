# 5.17 Signal Recovery Design: Reliability-Conditioned FC Generation

> BS-NET의 edge-level reliability map (ρ̂T)이 generative model 기반 signal recovery의 필수 전제(prerequisite)임을 실증하는 실험 설계.

---

## 1. 문제 정의

### 1.1 Reader의 Unmet Need

실제 연구 환경에서 대부분의 fMRI 데이터는 짧은 세션(3–5분 task scan 등)이다. 이 짧은 스캔에서 reliable FC를 추정하는 것이 핵심 과제.

### 1.2 현재 접근법의 한계

| 접근 | 방법 | 한계 |
|------|------|------|
| 직접 추정 | Short scan → correlation → FC | 0.01-0.1 Hz low-freq fluctuation이 undersampled. 60 TR (120s)에서 0.01 Hz = 1.2 주기만 관측. Edge별 SE ≈ 0.107 (vs long scan 0.033) |
| Generative recovery | Short scan → deep learning → predicted long FC | **Edge-level reliability 정보 없이 학습**. 모든 edge를 동등하게 취급 → reliable edge와 unreliable edge 구분 불가 → population average로 수렴 |
| BS-NET (현재) | Short scan → bootstrap + SB prophecy → ρ̂T | Reliability estimation은 제공하지만, **signal 자체를 복원하지 않음**. "이 edge가 얼마나 신뢰할 수 있는가"를 알려줄 뿐 |

### 1.3 핵심 가설

> **Reliability-conditioned generation > Reliability-unaware generation**
>
> Edge-level reliability map을 generative model에 conditioning으로 제공하면, reliability 없이 generation하는 것보다 FC estimation이 개선된다. BS-NET의 ρ̂T는 이 conditioning의 유일한 source이다.

### 1.4 타 분야 선례

Signal recovery에서 noise level estimation이 선행되어야 한다는 원칙은 fMRI 외 분야에서 이미 확립:

- **Image denoising**: Noise level estimation → denoising (Liu et al. 2013, Noise Level Estimation Using Weak Textured Patches)
- **Speech enhancement**: SNR estimation → noise reduction (Loizou 2007)
- **MRI reconstruction**: Noise map → compressed sensing / deep learning (Aja-Fernández et al. 2009)
- **fMRI FC 분야는 이 단계를 건너뛰고 있음** — reliability estimation 없이 바로 prediction/generation

## 2. 학계 현황 분석

### 2.1 FC Prediction/Generation 모델 (reliability 미사용)

| 모델 | 발표 | 접근 | Reliability 사용 | 코드 |
|------|------|------|-----------------|------|
| Guo et al. 2023 | Cerebral Cortex | GLM short→long FC prediction | ✗ | ✗ |
| Hierarchical Diffusion | ICLR 2025 | fMRI 시계열 합성 (hypergraph + multifractal) | ✗ | ✗ (미공개) |
| Spatiotemporal Coarse-to-Fine Diffusion | 2025 | ROI 시계열 → 2D → diffusion | ✗ | ✗ |
| T2I-Diff | MICCAI 2025 | Time-frequency BOLD → classifier-free diffusion | ✗ | ✗ |
| Brain-JEPA / Brain-Harmony | NeurIPS 2024/2025 | Self-supervised fMRI encoder (discriminative only) | ✗ | ✓ |
| **Diffusion-TS** | **ICLR 2024** | **Encoder-decoder transformer + Fourier decomposition** | **✗** | **✓** |

**공통점**: 어떤 모델도 edge-level reliability를 conditioning으로 사용하지 않음.

### 2.2 BS-NET의 위치

BS-NET은 유일하게 **edge-level reliability estimation**을 제공:
- Bootstrap ensemble → SB prophecy → ρ̂T (per-edge reliability)
- TSD ablation (E0–E2)에서 확인: BS-NET의 기존 파이프라인이 이미 최적 (implicit distillation 포화)
- **Signal recovery는 하지 않지만, signal recovery의 필수 전제를 제공**

## 3. 실험 설계

### 3.1 사용 모델: Diffusion-TS (ICLR 2024)

**선정 이유:**
- fMRI config 내장 (50 ROI BOLD time series)
- Imputation 모드 지원 (observed timepoints 고정 + missing 생성)
- **모델 변경 없이** test-time guidance로 conditional generation 가능
- Trend + Fourier decomposition이 BOLD low-freq 특성과 호환
- 코드 공개: https://github.com/Y-debug-sys/Diffusion-TS

**모델 구조 요약:**
- Encoder-Decoder Transformer (attention-based, not U-Net)
- Disentangled temporal representation: Trend (polynomial) + Seasonality (FFT Top-K) + Residual
- x_0 직접 예측 (noise prediction 아님)
- Loss: L2 + FFT loss (주파수 영역 reconstruction)
- Conditional sampling: Reconstruction-guided (Algorithm 1, gradient guidance)

### 3.2 데이터

**ds000243** (WashU resting-state, N=52):
- Full scan: 720 TR (30분, TR=2.5s)
- Short scan: 48 TR (120s) — 처음 48 TR 추출
- Schaefer-200 parcellation → 200 ROI time series

**Proof-of-concept 축소 (Phase 1):**
- 50 ROI subset (Diffusion-TS 원본 config와 동일 차원)
- Short: 48 TR → Long: 180 TR (450s, convergence peak 부근)

**Full-scale (Phase 2, Phase 1 성공 시):**
- 200 ROI, 48 TR → 180 TR

### 3.3 실험 조건

#### Condition A: Naive Imputation (baseline)

```
Short scan (48 TR) = observed
Missing (132 TR) = generated by Diffusion-TS (unconditional guidance)
→ Full imputed time series (180 TR)
→ FC_imputed = corr(imputed time series)
→ 평가: corr(FC_imputed, FC_reference)
```

Diffusion-TS의 reconstruction guidance (Algorithm 1):
- L1 = ||x_observed - x_hat_observed||² (all ROIs equally weighted)
- L2 = fluency loss

#### Condition B: Reliability-Guided Imputation (BS-NET conditioned)

```
Short scan (48 TR) → BS-NET → ρ̂T map [200×200] or [50×50]
ρ̂T를 ROI-level reliability로 변환: r_i = mean(ρ̂T[i, :])

Diffusion-TS reconstruction guidance 수정:
- L1 = || W_rho * (x_observed - x_hat_observed) ||²
- W_rho[i] = f(r_i): reliable ROI는 강한 constraint, unreliable ROI는 약한 constraint

→ Full imputed time series (180 TR)
→ FC_imputed_guided = corr(imputed time series)
→ 평가: corr(FC_imputed_guided, FC_reference)
```

**Modification point**: `engine/solver.py`의 `restore()` 메서드 내 guidance loss 계산 부분. 모델 자체는 변경 없음 (test-time guidance only).

#### Condition C: BS-NET Only (현재 파이프라인)

```
Short scan (48 TR) → BS-NET pipeline → ρ̂T
→ 평가: ρ̂T vs FC_reference correlation
```

### 3.4 평가 지표

| 지표 | 의미 | 축 |
|------|------|-----|
| r_FC | FC 행렬 간 Pearson correlation | Reference 일치 |
| Edge-level ICC | Per-edge intraclass correlation | 재현성 |
| Network ARI | Adjusted Rand Index of community structure | 네트워크 보존 |
| ρ̂T-stratified r_FC | Reliability tertile별 FC 일치도 | Reliability 의존성 |

### 3.5 기대 결과

| 조건 | 예상 r_FC | 근거 |
|------|----------|------|
| Raw short FC | ~0.716 | ds000243 E0 baseline (실측) |
| A (Naive imputation) | ~0.72-0.75 | Diffusion이 temporal extension하면 약간 개선 |
| **B (Reliability-guided)** | **~0.75-0.80** | **Reliable edge 보존 + unreliable edge 복원** |
| C (BS-NET only) | ~0.816 | ρ̂T (실측, 이미 확인) |

**핵심**: B > A가 증명되면 "reliability conditioning이 generative model을 개선한다" → BS-NET의 prerequisite 역할 실증.

**주의**: C > B일 가능성 높음 — BS-NET의 SB prophecy extrapolation이 diffusion generation보다 효과적일 수 있음. 이 경우에도 "B > A"만 성립하면 논문의 contribution은 유효.

## 4. 구현 계획

### Phase 1: Proof-of-Concept (50 ROI, ds000243)

1. **데이터 준비**: ds000243에서 50 ROI subset 추출 (Schaefer-200의 상위 50 ROI)
   - Short: 48 TR, Long: 180 TR, Full: 720 TR
   - Train/test split: 42/10 subjects

2. **Diffusion-TS 학습**: fMRI config 기반, 50 ROI × 180 TR
   - 학습 데이터: full scan에서 random crop (180 TR windows)
   - ~15,000 steps, ~1h on single GPU

3. **BS-NET 실행**: 50 ROI subset에 대해 ρ̂T 산출

4. **Imputation 실험**: Condition A vs B vs C 비교

5. **평가**: r_FC, edge-level ICC, ARI

### Phase 2: Scale-Up (200 ROI, Phase 1 성공 시)

- Diffusion-TS config 조정: d=200, seq_len=180
- Memory 최적화: batch_size 축소, gradient checkpointing
- Multi-dataset validation: ds000243 + ABIDE subset

### Phase 3: 논문 반영

- Methods: "Reliability-Conditioned Signal Recovery" 섹션
- Discussion: "BS-NET as prerequisite for generative FC recovery"
- Figure: Condition A vs B vs C 비교 (bar plot + scatter)

## 5. 코드 수정 명세

### 5.1 Diffusion-TS 수정 (최소 변경)

**파일**: `engine/solver.py` — `restore()` 메서드

현재 (reconstruction guidance, Algorithm 1):
```python
# L1: conditional consistency (all channels equal)
loss_obs = F.mse_loss(x_hat_obs, x_obs)
```

수정 (reliability-weighted):
```python
# W_rho: ROI-level reliability weight from BS-NET
# shape: [batch, 1, n_roi] — broadcast over time dimension
loss_obs = (w_rho * (x_hat_obs - x_obs) ** 2).mean()
```

**파일**: `Config/fmri_bsnet.yaml` — 신규 config
```yaml
model:
  params:
    seq_length: 180
    feature_size: 50      # Phase 1
    n_layer_enc: 4
    n_layer_dec: 4
    d_model: 96
    timesteps: 1000
```

### 5.2 BS-NET → Diffusion-TS 데이터 브릿지

```python
# bs-net output: per-edge reliability
rho_hat_T = np.load("rho_hat_T_50roi.npy")  # [50, 50]

# Edge-level → ROI-level aggregation
roi_reliability = rho_hat_T.mean(axis=1)  # [50]

# Normalize to [0, 1] weight
w_rho = (roi_reliability - roi_reliability.min()) / (roi_reliability.max() - roi_reliability.min())

# Save for Diffusion-TS
np.save("reliability_weight.npy", w_rho)
```

## 6. 리스크 및 대안

| 리스크 | 확률 | 대안 |
|--------|------|------|
| 50→200 ROI 확장 실패 | 중간 | Phase 1의 50 ROI 결과만으로 논문 proof-of-concept |
| B ≈ A (reliability conditioning 효과 없음) | 낮음 | Edge-level 직접 conditioning (ROI-level 대신) |
| Diffusion 생성 FC < Raw short FC | 낮음 | Training 데이터/hyperparameter 조정 |
| C >> B (BS-NET이 압도적) | 높음 | **이것도 긍정적 결과**: "BS-NET의 analytic approach가 generative approach보다 efficient" |
