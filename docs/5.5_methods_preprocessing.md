# 5.5 Preprocessing Pipeline — Methods Section Draft

## 5.5.1 Data Acquisition

Resting-state functional MRI (rsfMRI) data were obtained from 7 publicly
available OpenNeuro datasets comprising healthy adult controls. Acquisition
parameters varied across datasets (see Supplementary Table S1). In general:

- **Repetition time (TR)**: 1.0–2.5 s (dataset-dependent)
- **Total scan duration**: ≥5 minutes per subject (reference FC computed from full scan)
- **Short-scan evaluation**: First 2 minutes (120 s) extracted for BS-NET input
- **Spatial resolution**: 2–3 mm isotropic (dataset-dependent)

## 5.5.2 Preprocessing: fMRIPrep

All functional images were preprocessed using **fMRIPrep v24.1.1**
(Esteban et al., 2019; DOI: 10.1038/s41592-018-0235-4) with the following
key settings:

1. **Head motion correction** — 6-parameter rigid-body realignment.
   Framewise displacement (FD) computed for each volume.
2. **Susceptibility distortion correction** — fieldmap-based when available;
   fieldmap-less (SyN-based) as fallback.
3. **Spatial normalization** to **MNI152NLin6Asym** template at 2 mm
   resolution via nonlinear registration (ANTs).
4. **Brain extraction** using ANTs-based brain masking.
5. **BIDS validation skipped** (`--skip-bids-validation`) for OpenNeuro
   compatibility; fieldmaps and slice timing correction ignored
   (`--ignore fieldmaps slicetiming`) as not all datasets provide these.

fMRIPrep outputs: MNI-space preprocessed BOLD (`*_desc-preproc_bold.nii.gz`),
confounds TSV (`*_desc-confounds_timeseries.tsv`), and brain mask.

## 5.5.3 Post-Processing: XCP-D

Denoising was performed using **XCP-D v0.10.0**
(Ciric et al., 2017; DOI: 10.1016/j.neuroimage.2017.08.002;
Adebimpe et al., 2023; DOI: 10.1038/s41592-023-01917-3)
applied to fMRIPrep outputs:

- **Confound regression**: 36-parameter model (36P) —
  6 motion parameters + temporal derivatives + quadratic terms (24 motion),
  plus mean white matter and CSF signals + derivatives + quadratic terms
  (8 tissue), plus 4 global signal regressors.
- **Despiking**: Applied prior to confound regression (`--despike`).
- **Bandpass filtering**: 0.01–0.1 Hz.
- **Motion scrubbing**: Volumes with FD > 0.5 mm censored.
  Subjects with <100 s of remaining data after scrubbing excluded
  (`--min-time 100`).
- **Spatial smoothing**: 6 mm FWHM Gaussian kernel.
- **Parcellation**: Schaefer 100-parcel and 400-parcel atlases
  (7-network assignment) applied simultaneously via `--atlases Schaefer100 Schaefer400`.

XCP-D outputs per subject: parcellated time series TSV
(`*_atlas-Schaefer{100,400}_timeseries.tsv`) and correlation matrices
(`*_atlas-Schaefer{100,400}_relmat.tsv`).

## 5.5.4 FC Computation for BS-NET

For BS-NET evaluation, functional connectivity was computed from
XCP-D parcellated time series (post-censoring):

1. **Short-scan FC**: First 2 minutes of the cleaned time series.
   Covariance estimated via **Ledoit-Wolf shrinkage**
   (Ledoit & Wolf, 2004; DOI: 10.1016/j.jmva.2004.02.003),
   converted to correlation matrix.
   - Shrinkage is critical when `n_timepoints < n_parcels`
     (e.g., ~60–120 volumes < 400 parcels for the short scan).
   - Sensitivity analysis (Track D) confirmed negligible performance
     difference between Ledoit-Wolf and Oracle Approximating Shrinkage
     (OAS; Chen et al., 2010).
2. **Reference FC**: Computed from the full-length cleaned time series
   using Ledoit-Wolf shrinkage (same estimator for consistency).

Parcellation granularity:
- **Schaefer 100**: Primary analysis (simulation and real data).
- **Schaefer 400**: Scalability demonstration (simulation N=400 ROI;
  real data pending full XCP-D processing).

## 5.5.5 Quality Control

- **Motion criteria**: Subjects with mean FD > 0.3 mm or maximum FD > 5 mm
  excluded prior to processing.
- **Post-scrubbing duration**: Subjects with <100 s effective scan time
  after XCP-D censoring excluded.
- **Signal quality**: ROI time series with zero variance flagged;
  subjects excluded if >10% of parcels affected.
- **Coverage check**: Parcels with <50% overlap with the brain mask
  excluded from analysis.

## 5.5.6 BS-NET Pipeline Parameters

The BS-NET pipeline uses the following default parameters, each with
empirical justification:

| Parameter | Default | Justification |
|-----------|---------|---------------|
| `reliability_coeff` | 0.98 | Within-session scanner reliability (Friedman et al., 2008; DOI: 10.1016/j.neuroimage.2008.02.005, ICC > 0.95) |
| `n_bootstraps` | 100 | Sufficient for stable percentile CI estimation |
| `empirical_prior` | (0.25, 0.05) | Population-level split-half reliability mean from HCP data (Cheng et al., 2021; DOI: 10.1016/j.neuroimage.2021.118005) |
| `k_factor` | 7.5 | Spearman-Brown scaling: 15 min / 2 min |
| Block length | Adaptive | Estimated from AR(1) structure per Politis & White (2004) heuristic |
| CI method | Percentile (2.5th–97.5th) | Applied in Fisher-z space, back-transformed to correlation scale |

## 5.5.7 Statistical Analysis

- All correlations were Fisher-z transformed before aggregation and
  confidence interval computation, then back-transformed for reporting.
- Bootstrap confidence intervals use the **percentile method** (2.5th and
  97.5th percentiles of the Fisher-z bootstrap distribution).
- FC agreement between predicted and reference FC was quantified using
  Pearson correlation of vectorized upper-triangle FC matrices (denoted rFC).
- Pipeline output (extrapolated reliability) is denoted ρ̂T.
