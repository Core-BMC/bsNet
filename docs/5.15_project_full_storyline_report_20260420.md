# 5.15 BS-NET 프로젝트 전체 스토리라인 리포트 (2026-04-20)

## 1) 리포트 목적
이번 리포트는 최근 Keane 실험 단일 결과가 아니라, 현재까지의 전체 프로젝트 흐름(설계 의도 → 실험 실행 → 결과 → 해석)을 한 문서로 정리해 다음 의사결정을 위한 기준을 제공한다.

핵심 질문은 두 가지였다.

1. BS-NET이 짧은 rsfMRI에서 신뢰도(reliability)를 안정적으로 추정/보정하는가  
2. 그 신뢰도 보정이 실제 downstream 결론(특히 환자 discrimination)을 바꾸는가

---

## 2) 프로젝트 출발점과 방법론 의도

### 문제 인식
- 짧은 스캔(약 2분)은 실용적이지만 FC 안정성이 낮다.
- 현장에서는 “짧은 스캔으로도 해석 가능한 신뢰도”가 필요하다.

### BS-NET 파이프라인 설계
- Ledoit-Wolf shrinkage FC
- block bootstrap resampling
- Spearman-Brown prophecy (`k`)
- Bayesian empirical prior
- attenuation correction
- (후속 핵심 변경) Fisher z-space correction

의도는 “짧은 관측으로부터 긴 관측 수준 신뢰도(\rho_hat_T)를 외삽”하는 것이었다.

## 5) 주요 실험 결과 요약

## 5.1 ABIDE/ADHD에서 reliability 축

### ABIDE PCP (N=468, Fisher z)
- `r_FC = 0.771 ± 0.071`
- `rho_hat_T = 0.843 ± 0.036`
- ceiling 0%

### ABIDE CC400
- `rho_hat_T = 0.834 ± 0.037`
- atlas 규모 증가에도 성능 동등 수준

### ADHD-200 PCP full (N=399)
- `r_FC = 0.525 ± 0.087`
- `rho_hat_T = 0.725 ± 0.049`
- 전반 개선 일관

해석:
- BS-NET은 여러 데이터셋/atlas 조건에서 reliability estimator로는 일관 작동.

---

## 5.2 Convergence / τ_min 축
- ds000243에서 duration sweep + 다중 seed 검증
- `rho_hat_T(τ)` plateau 기반 `τ_min` 경험적 범위 제시
- 실용 권고 범위: 60–120s (plateau: 90–180s)

해석:
- “왜 짧은 스캔에서도 신뢰도 추정이 가능한가”에 대한 방법론적 근거를 강화.

---

## 5.3 ADHD utility(클러스터링/분류)
- reliability strata를 이용한 비지도/감독 분석 수행
- 일부 조건에서 약한 신호는 있으나, 임상 분리 성능의 강한 일관 개선은 제한적
- permutation 검정 포함 시 유의성은 더 약화

해석:
- reliability 개선이 discrimination 개선으로 자동 전이되지 않음.

### 1차 분류

- split validity, coverage, permutation, Holm/FDR 포함

해석:
- Keane BP vs SZ에서 BS-NET 기반 gating이 유의한 discrimination 개선을 입증하지 못함.

## 6) Keane gated 분석 중 디버깅/개선 이력
실험 중 반복적으로 발생한 “NaN/invalid split” 문제를 다음과 같이 해결했다.

1. train/test 최소 클래스 기준 분리  
2. hard gate threshold를 `nanquantile` + finite mask로 안정화  
3. non-finite `rho_hat_T` 사전 제거  
4. `min-valid-splits` 도입  
5. invalid setting은 permutation 스킵  
6. summary에 `invalid_rate` 반영  
7. 진행률(tqdm) 및 경고 로그 개선

의미:
- 현재 코드는 “실패 원인이 모델 이론인지/설정 불가능성인지”를 구분해서 보고 가능.

## 7) 전체 스토리라인 관점의 통합 해석

### 확인된 것
1. BS-NET은 reliability 추정/보정 도구로 일관되게 작동한다.  
2. 수학적/구현적 안정성(특히 Fisher z 보정)은 충분히 강화되었다.

### 확인되지 않은 것
1. 현재 코호트/태스크에서 discrimination enhancer로서 유의한 효능  
2. reliability 측정 자체가 곧바로 최종 결론 변화로 이어진다는 실증

### 현재 가장 정확한 평가
- “방법론 성립은 확인”  
- “실용 효능(결론 변화)은 미입증”

## 11) 최종 결론 (요약)
이번까지의 전 과정은 기술적으로 높은 완성도로 수행되었고, BS-NET의 reliability 추정 메커니즘은 다각도로 검증되었다.  
그러나 현재 데이터/태스크 조건에서 “분류 성능 향상”이라는 실질 효능은 통계적으로 입증되지 않았다.  
따라서 현 시점의 가장 정직한 결론은 다음과 같다.

- BS-NET은 **작동하는 방법론**이다.  
- 하지만 지금 조건에서 **강한 실용 효능 주장은 보류**가 타당하다.
